# Processus DEV M

## 1. Développement Agile (Scrum)

```
Backlog → Sprint Planning → Développement → Daily → Review → Rétro
```

| Rituel | Fréquence | Durée | Participants |
|--------|-----------|-------|--------------|
| Sprint Planning | Début sprint | 2h | Toute l'équipe |
| Daily Standup | Quotidien | 15 min | Équipe dev |
| Sprint Review | Fin sprint | 1h | Équipe + PO |
| Rétrospective | Fin sprint | 1h | Équipe dev |

## 2. Cycle de Développement

```
Feature → Branche → Dev → Tests → Review → Merge → CI/CD → Déploiement
```

| Étape | Responsable | Durée |
|-------|-------------|-------|
| Spécification | Chef de Projet | — |
| Développement | Développeur | 1-3 jours |
| Tests unitaires | Développeur | Intégré |
| Code Review | Lead Tech | < 24h |
| CI/CD | Automatisé | ~10 min |

## 3. Maintenance Applicative

- Correction de bugs (priorité selon sévérité)
- Mise à jour des dépendances (Dependabot)
- Veille technologique
